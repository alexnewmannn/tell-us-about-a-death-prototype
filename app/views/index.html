Changes

Following several co-creation workshops and paper prototyping sessions with telephony colleagues, this represents the first digital iteration of the Agent-facing MVP.

Features

  • Navigation for capturing New notification and for finding an existing notification
  • New notification data capture features tabs to separate content
  • ‘Check answers’ page allows the Agent to go back and edit content as required before submitting
  • ‘Find notification’ feature purpose for after-call work only at this stage – intentionally designed to be read-only

Hypotheses

  • Users will prefer a flexible approach to data capture – reflecting the organic nature of telephone calls.

Research notes

  • ‘New notification’ flow tested with Agents from both TUO and DWP Bereavement services at Tyne View Park, 02/05/2018.

Feature
End Verify
Related stories

Changes

In this version, we created a prototype to reflect recent conversations around security; specifically trust and disclosure. We placed GOV Verify links at the end, together with the option to take a unique reference number to a Registrar to prove identity prior to receiving playback from the system.

Changes from previous versions;

  • Start page features soft intro paragraph, exanded list of government services and actions, & a note on proving identity later in the process
  • Relationship to deceased status is an open text field
  • Notification complete stage features a unique reference number
  • The end list of system actions is hidden behind mandatory proof of identity methods

Hypotheses

  • Users would prefer the open text field for setting relationship status
  • People might abandon process at Verify stage even though they expect/want to receive disclosure at the end.

Changes

These sketched prototypes follow on from the UR work carried out in Newcastle library. Tested with Registrar's, this version features the intentionally bloated relationship status data ask.

Changes to the previous sketches;

  • Legal declaration on the start page
  • Extended list visual for relationship status
  • Disclosure list features 4 sections, date, and print page

Hypothesis

We were looking to establish whether the legal declaration would have an impact on users. We also wanted to provoke a discussion around relationship status - hence the intentionally unusable list shown in the prototype.

That access to other department's technology will allow us to feed back to the citizen in a more tailored way. And that they should have the ability to amend the list of services if they see an omission.
